# Rita2API Docs 导航

## 1. 文档说明

这里汇总本项目当前与账号体系相关的文档，方便从 `docs` 目录直接开始阅读，而不是先回到仓库根目录找入口。

如果你是第一次接触这个仓库，建议先从“账号体系总览”开始，再按专题继续下钻。

## 2. 推荐阅读顺序

### 2.1 第一步：先看总览

- [账号体系总览](./账号体系总览.md)

适合场景：

- 想先知道这套系统整体在干什么

### 2.2 第二步：看注册主链

- [注册整体流程](./注册整体流程.md)

适合场景：

- 想看账号怎么注册出来
- 想排查邮箱、验证码、captcha、token、ticket、激活链路
- 想知道新号是怎么落库和初次验活的

### 2.3 第三步：看请求如何消耗账号池

- [注册后聊天请求轮换流程](./注册后聊天请求轮换流程.md)

适合场景：

- 想看 `/v1/chat/completions`、`/v1/responses` 怎么选号
- 想看 round-robin、失败计数、扣点、conversation state
- 想看新注册账号为什么不一定下一条就被打到

### 2.4 第四步：看运行期维护链

- [健康检查、自动补号、刷新 Token 联动流程](./健康检查自动补号刷新Token联动流程.md)

适合场景：

- 想看坏号怎么被踢出池子

## 5. 建议维护方式

后续如果账号体系继续扩展，建议按下面规则维护：

1. **先更新专题文档**
   - 例如新增 refresh 分支、补号逻辑、账号池规则，先改对应专题页
2. **再更新总览页**
   - 如果结构变化影响整体认知，再改 `账号体系总览.md`
3. **最后更新本目录页**
   - 如果新增了新的专题文档，再把入口补到 `docs/README.md`

这样可以保持：

- 专题页负责细节
- 总览页负责结构
- 目录页负责导航
