# 远程账号同步设计（本地手动同步到远程 rita2api）

## 1. 背景

当前 `rita2api` 已支持本地账号注册、账号管理、批量导入与批量操作。  
用户现在需要在**本地服务**完成注册后，将账号**手动同步**到另一台已部署的**远程 `rita2api` 服务**。

第一版目标是：

- 在本地“账号管理”页发起同步
- 支持“同步已选账号”与“同步全部账号”
- 远端按 `email` 去重
- `quota_remain < 10` 的账号不参与同步
- 无 `email` 的账号不参与同步
- 已存在账号不覆盖、不更新，只跳过

## 2. 约束与原则

- 保持**最小改动**
- 复用现有 `templates/index.html` 单文件管理面板
- 复用现有 `/api/accounts`、`/api/accounts/batch` 管理链路
- 不把远程管理口令暴露给浏览器跨域请求
- 不引入多目标管理，不落库存远程目标
- 第一版只做“新增同步”，不做远端覆盖更新

## 3. 方案选择

### 方案 A（采用）

本地前端调用本地后端新接口，由**本地后端**去请求远程服务。

链路：

1. 用户在本地账号管理页点击“同步到远程”
2. 前端把“远程地址 + 远程管理 API + 同步范围”发给本地 `/api/accounts/sync-remote`
3. 本地后端读取本地 SQLite 真实账号数据
4. 本地后端请求远端 `/api/accounts`
5. 本地后端按规则过滤并去重后，请求远端 `/api/accounts/batch`
6. 前端展示同步结果摘要

### 方案 B（不采用）

前端直接跨域请求远端 `/api/accounts/*`。

不采用原因：

- 现有服务未显式处理 CORS
- 远程管理口令会进入浏览器请求链路
- 与现有管理接口的鉴权边界不一致

### 方案 C（暂不采用）

远端新增专用 `/api/accounts/remote-import`。

暂不采用原因：

- 第一版无需新协议面
- 相比复用现有导入接口，改动更重

## 4. 前端设计

位置：沿用“账号管理”页。

新增内容：

1. 页头新增按钮：`同步到远程`
2. 批量工具栏新增按钮：`同步远程`
3. 新增同步弹窗，包含：
   - 远程服务地址
   - 远程管理 API Key（实际走远端 `AUTH_TOKEN`）
   - 同步范围：
     - 已选账号
     - 全部账号
4. 新增结果区域，展示：
   - 扫描总数
   - 实际同步新增数
   - 因无邮箱跳过数
   - 因积分不足跳过数
   - 因远端已存在跳过数
   - 因本地重复邮箱跳过数
   - 失败数

前端仅把**范围和远端鉴权信息**提交给本地后端，不直接携带账号 token 列表，避免把完整账号敏感字段暴露给浏览器脚本层。

## 5. 后端设计

新增接口：

- `POST /api/accounts/sync-remote`

请求体：

```json
{
  "remote_base_url": "https://example.com",
  "remote_auth_token": "xxxx",
  "scope": "selected",
  "ids": ["id1", "id2"]
}
```

其中：

- `scope` 支持 `selected` / `all`
- `scope=selected` 时要求传 `ids`

后端执行步骤：

1. 校验参数
2. 从本地数据库取出完整账号
3. 按规则过滤：
   - `email` 为空 -> 跳过
   - `quota_remain < 10` -> 跳过
4. 本地按 `email.lower().strip()` 去重，只保留第一条
5. 请求远端 `GET /api/accounts`
6. 生成远端已有邮箱集合
7. 过滤掉远端已存在邮箱
8. 调用远端 `POST /api/accounts/batch`
9. 返回汇总结果

## 6. 导入字段兼容

为了让远端保留本地真实账号状态，需要扩展现有导入接口对以下字段的可选支持：

- `quota_remain`
- `enabled`

兼容原则：

- 老调用不传时，仍沿用当前默认值
- 新同步功能传入时，远端按传入值落库

## 7. 去重与过滤合同

### 远端去重

- 仅按 `email` 精确匹配（忽略大小写、去首尾空格）
- 命中则跳过，不更新远端既有记录

### 本地过滤

- `email == ''` -> 跳过
- `quota_remain < 10` -> 跳过

### 本地重复邮箱

- 同步时若本地多条账号 `email` 相同，只保留第一条
- 其余记录记入“本地重复跳过”

## 8. 错误处理

- 远端地址不可达 / 超时 -> 整次同步失败
- 远端鉴权失败 -> 返回明确错误
- 远端拉取账号失败 -> 整次同步失败
- 远端批量导入失败 -> 返回远端错误信息
- 若无可同步账号 -> 返回成功但新增数为 0，并附原因统计

不做：

- 不做事务回滚
- 不做远端删除
- 不做远端覆盖更新

## 9. 验证方案

最小验证：

1. Python 语法校验
2. 前端脚本语法校验
3. Flask test client / mock 远端接口验证以下场景：
   - selected 同步
   - all 同步
   - 无邮箱跳过
   - 积分不足跳过
   - 远端重复跳过
   - 本地重复跳过
   - 导入成功保留 `quota_remain`
